## ADDED Requirements

### Requirement: Peer Skill Files Exist
The repository SHALL provide equivalent `repo-scaffold` skill definitions for Codex and Claude Code.

#### Scenario: Skill Is Located
- **WHEN** a contributor inspects `.codex/skills/repo-scaffold/` and `.claude/skills/repo-scaffold/`
- **THEN** each directory contains a real `SKILL.md` for the repo-scaffold skill (not just a `.gitkeep`).

### Requirement: Skill Accepts Approved Syllabus And Manifests
The repo-scaffold skill SHALL accept the approved syllabus YAML and all per-lesson manifests (with populated `outline`, `sources`, and optional `framing_notes`) as its required inputs.

#### Scenario: Inputs Are Provided
- **WHEN** the skill receives the approved syllabus and per-lesson manifests from the content-planner digest
- **THEN** it begins writing files without asking for scoping, shape, or curriculum information again.

### Requirement: Skill Writes LEARNING.md And STYLE.md
The repo-scaffold skill SHALL write `LEARNING.md` containing topic, goal, framing, shape, build piece, and scoping answers, and SHALL write a finalized `STYLE.md` incorporating any preferences expressed during scoping.

#### Scenario: Topic Files Are Written
- **WHEN** the skill scaffolds the repository
- **THEN** `LEARNING.md` exists and matches the §5.1 schema, and `STYLE.md` reflects the scoping conversation preferences.

### Requirement: Skill Writes OpenSpec Bridge Files
The repo-scaffold skill SHALL write `openspec/project.md` with topic, goal, framing, shape, tech stack, conventions, and the full syllabus YAML; and SHALL write `openspec/AGENTS.md` with lesson-implementation rules including directory conventions, required file set, test additivity rule, and koan/build-piece task templates.

#### Scenario: OpenSpec Bridge Files Are Written
- **WHEN** the skill scaffolds the repository
- **THEN** `openspec/project.md` exists matching the §5.7 schema and `openspec/AGENTS.md` exists with lesson implementation rules.

### Requirement: Skill Creates Per-Lesson Change Directories Using Correct Template
The repo-scaffold skill SHALL create `openspec/changes/{lesson-id}/` for every lesson in the syllabus, selecting Template A (stable koan), Template B (versioned/frontier koan), or Template C (build piece) based on the lesson's `type` and `currency` fields.

#### Scenario: Stable Koan Lesson Is Scaffolded
- **WHEN** a lesson has `type: koan` and `currency: stable`
- **THEN** its change directory contains `proposal.md` and `tasks.md` (Template A — no `design.md`).

#### Scenario: Versioned Koan Lesson Is Scaffolded
- **WHEN** a lesson has `type: koan` and `currency: versioned` or `currency: frontier`
- **THEN** its change directory contains `proposal.md`, `design.md` (the content manifest), `tasks.md`, and `sources.md` (Template B).

#### Scenario: Build-Piece Lesson Is Scaffolded
- **WHEN** a lesson has `type: build_piece`
- **THEN** its change directory contains `proposal.md`, `design.md`, `specs/` delta, and `tasks.md` (Template C), regardless of currency.

### Requirement: Skill Writes Research Prompts For High-Audio Lessons
The repo-scaffold skill SHALL write `research-prompts/{lesson-id}.md` for every lesson with `audio_value: high`, using the prompt content generated by the research-prompt-generator during the bulk planning pass.

#### Scenario: High-Audio Lesson Is Scaffolded
- **WHEN** a lesson has `audio_value: high` and a research prompt was generated during planning
- **THEN** `research-prompts/{lesson-id}.md` is written matching the §5.6 schema.

### Requirement: Skill Creates Empty Lessons Directory
The repo-scaffold skill SHALL create an empty `lessons/` directory as a placeholder for lesson implementations added by `/opsx:apply`.

#### Scenario: Lessons Directory Exists After Scaffold
- **WHEN** the skill completes scaffolding
- **THEN** a `lessons/` directory exists (may be empty or contain a `.gitkeep`).

### Requirement: Skill Makes Initial Git Commit
The repo-scaffold skill SHALL stage all written files and create an initial commit with the message `Bootstrap learning project: {topic}`.

#### Scenario: Initial Commit Is Created
- **WHEN** the skill finishes writing all files
- **THEN** a git commit exists with message `Bootstrap learning project: {topic}` containing all scaffolded files.

### Requirement: Skill Reports Scaffold Summary
The repo-scaffold skill SHALL print a summary after completing that includes the lesson count, number of files written, and the next-step instruction (`/opsx:apply lesson-01-...`).

#### Scenario: Summary Is Printed
- **WHEN** scaffolding completes
- **THEN** the skill outputs the lesson count, total files written, and instructs the learner to run `/opsx:apply` on the first lesson.

### Requirement: Skill Does Not Generate Lesson Content
The repo-scaffold skill SHALL NOT write `lesson.md`, test files, or workspace stub files inside `lessons/`. Those are written by `/opsx:apply` when each lesson is implemented.

#### Scenario: Lesson Content Is Absent After Scaffold
- **WHEN** the skill completes scaffolding
- **THEN** the `lessons/` directory contains no lesson content files — only the directory itself (and optionally a `.gitkeep`).
